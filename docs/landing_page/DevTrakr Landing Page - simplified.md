# DevTrakr Landing Page 功能分析

DevTrakr 的定位不是泛化的"项目管理工具"，而是面向研发团队的 **AI 驱动型 Issue 管理与研发协同平台**。核心价值是把 Issue、GitHub 仓库、团队协作、AI 分析串成一条闭环。

## 产品定位

| 维度 | 结论 |
|---|---|
| 产品本质 | 面向研发团队的 Issue 驱动协作平台 |
| 核心用户 | 产品经理、开发、测试、团队管理员 |
| 差异化 | AI 分析 + GitHub 联动 + 团队协作闭环 |
| Landing 类型 | B2B SaaS / 企业研发效率工具 |
| 首屏目标 | 立刻让用户理解这是"研发问题处理中枢",不是普通看板 |

## 三条主价值线

- **Issue 闭环管理**:问题从创建、分析、处理到复盘全在一个系统内完成
- **研发上下文联动**:Issue 与 GitHub 仓库、PR、commit 不再割裂
- **AI 驱动洞察**:AI 不是聊天,而是面向研发问题的分析层(原因、方案、瓶颈、趋势)

建议首屏主卖点文案:

> DevTrakr 是面向研发团队的 AI 驱动 Issue 管理平台,把问题跟踪、GitHub 上下文和团队洞察整合到一个工作台中。

## 页面结构(按转化路径排布)

| 模块 | 内容 |
|---|---|
| Hero 首屏 | 产品定位 + 主/次 CTA |
| 痛点共鸣 | Issue 分散、GitHub 上下文割裂、协作断层、缺少全局洞察 |
| 能力 1:Issue 工作台 | 看板/列表、筛选、批量处理、责任分配 |
| 能力 2:GitHub 联动 | 仓库接入、Issue/PR/commit 关联 |
| 能力 3:AI 分析洞察 | 原因分析、解决建议、趋势预警 |
| 协作与治理 | 通知、权限、用户管理、API |
| 使用流程 | 接入仓库 → 同步 Issue → AI 分析 → 协作推进 → 复盘 |
| 最终 CTA | 申请试用 / 预约演示 |

## 功能优先级

| 优先级 | 功能 |
|---|---|
| P0 | Hero 标题副标题、主/次 CTA、三段式能力展示、真实界面截图、角色场景、FAQ |
| P1 | 团队收益数据、架构/集成说明、证言占位 |
| P2 | 多语言、交互动效 |

## 视觉提醒

深色 + 企业级 + AI 科技感 调性匹配,但首屏不能只靠大字和抽象背景,必须有一句明确的价值句,避免被误认为"大模型官网"。
